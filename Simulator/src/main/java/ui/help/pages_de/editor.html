Modell-Editor

Der Modell-Editor umfasst 7 Seiten, über die alle Parameter des zu simulierenden Callcenter-Modells eingestellt werden können:

Modell

Auf dieser Seite können keinen Einstellungen vorgenommen werden, sondern es wird lediglich das im Simulator abgebildete Simulationsmodell dargestellt.

In der Simulation können folgende Eigenschaften eines realen Warteschlangen bzw. Callcenter-Systems abgebildet werden:

  • Beliebige Verteilung der Zwischenakunftszeiten
  • Gruppenankünfte
  • Warteraumbegrenzungen
  • Begrenzte Wartezeittoleranzen der Kunden
  • Wiederholwahrscheinlichkeiten und Wiederholabstände der Warteabbrecher
  • Eine beliebige Anzahl an Agenten
  • Bedienreihenfolge FIFO oder LIFO
  • Beliebige Verteilung der Bedienzeiten
  • Gruppenbedienungen
  • Weiterleitungswahrscheinlichkeit nach einer Bedienung

Folgende ebenfalls relevante Eigenschaften eines realen Callcenter-Systems kann dieser Mini Callcenter Simulator jedoch nicht abbilden:

  • Transiente Phase, d.h. Veränderung der Ankunftsrate und der Agentenanzahl über den Tag
  • Verschiedene Kundentypen mit verschiedenen Eigenschaften
  • Verschiedene Agententypen mit verschiedenen Skill-Leveln
  • Priorisierungsregeln für bestimmte Kunden- und Agentengruppen

Der kommerzielle Callcenter Simulator kann all diese Eigenschaften abbilden und besitzt darüber hinaus wesentlich umfangreichere Werzeuge zur Auswertung von Simulationsläufen und zur teilautomatischen Optimierung von Callcenter-Systemen.

Allgemeines

Auf der Seite "Allgemeines" können ein Name und eine Beschreibung für das Modell angegeben werden. Diese Daten werden in der Simulation nicht weiter verwendet, werden aber zusammen mit dem Modell bzw. zusammen mit den Statistikergebnissen gespeichert, so dass später nachvollzogen werden kann, was mit dem jeweiligen Modell untersucht werden sollte.

Über das Feld Zu simulierende Anrufe kann ferner eingestellt werden, wie viele virtuelle Kunden in der Simulation betrachtet werden sollen. Je höher die hier eingestellte Anzahl ist, desto weniger schwanken die Simulationsergebnisse aber auch desto länger dauert die Simulation. Da in der Simulation die stationäre Phase des Warteschlangensystems abgebildet werden soll, wird vor dem Beginn der Zählung der Anrufe und der Erfassung der Statistikergebnisse eine Einschwingphase eingefügt. Die Einschwingphase dauert 10% so lange wie die spätere eigentliche Simulation.

Ankünfte

Auf der Seite "Ankünfte" kann über das Feld Kunden pro Ankunft festgelegt werden, wie viele Kunden pro Ankunftsereignis eintreffen. Soll ein Callcenter-System, in dem die Kunden einzelnen eintreffen, modelliert werden, so kann dieser Wert auf 1 belassen werden. Über den Verteilung der Zwischenankunftszeiten Wahrscheinlichkeitseditor kann festgelegt werden, in welchem Abstand die Ankünfte stattfinden. Treffen Kunden in einem System ein, ohne ihre Ankünfte gegenseitig wahrzunehmen (wie dies z.B. in einem Callcenter-System der Fall ist), so eignet sich zur Modellierung der Zwischenankunftszeiten die Exponentialverteilung.

Warteraum und Wartezeittoleranz

Über die Warteraumbegrenzung kann eingestellt werden, wie viele Kunden sich maximal gleichzeitig in der Warteschlange befinden können. Kunden, die bereits bedient werden, belegen keinen Platz in der Warteschlange, d.h. eine Warteraumbegrenzung bezieht sich nur auf die wartenden Kunden. Trifft ein Kunde ein und ist kein freier Warteplatz verfügbar, so wird dieser abgewiesen und verlässt unmittelbar das System. Er wird als Warteabbrecher mit einer Wartezeit von 0 Sekunden erfasst. Genauso wie Kunden, die aufgrund einer zu langen Wartezeit das Warten vorzeitig aufgegeben haben, kann auch solch ein erzwungener Warteabbrecher später evtl. einen weiteren Anrufversuch starten.

Ist eingestellt, dass die Kunden nur eine begrenzte Wartezeittoleranz besitzen und nicht bereit sind, beliebig lange zu warten, so brechen die Kunden das Warten ab (und legen z.B. im Falle eines Callcenters auf), wenn ihre individuelle Wartezeittoleranz überschritten wurde. Die Wartezeittoleranz ist kein für alle Kunden konstanter Wert, sondern wird individuell pro Kunde gemäß der im Verteilungseditor auf dieser Seite eingestellten Wartezeittoleranzverteilung bestimmt. Die Wartezeittoleranzverteilung kann sehr verschiedene Verteilungstypen annehmen und ist in der Praxis meist schwer zu erheben.

Die Erfassung der Autokorrelation der Wartezeiten verlangsamt die Simulation, ermöglicht aber zu sehen, wie stark der Einfluss der Wartezeit eines Kunden auf die Wartezeiten der folgenden Kunden ausfällt.

Bedienungen

Auf der Seite "Bedienungen" können alle für den Bedienprozess relevanten Parameter eingestellt werden. Über das Eingabefeld Anzahl an Agenten kann festgelegt werden, wie viele Agenten parallel in dem System arbeiten und Kunden bedienen.

Die Bedienreihenfolge gibt an, welcher der wartenden Kunden jeweils als nächstes bedient werden soll. Die Bedienung kann in Ankunftsreihenfolge (FIFO) oder in umgekehrter Ankunftsreihenfolge (LIFO) erfolgen. Eine FIFO-Bedienung entspricht dem Prinzip einer Warteschlange. Eine LIFO-Bedienung tritt immer dann auf, wenn die wartenden Werkstücke gestapelt werden und der Stapel nur von oben befüllt und Werkstücke wieder entnommen werden können.

Im Callcenter-Bereich bearbeitet jeder Agent jeweils einen Kunden; im Kontext allgemeinerer Bediensysteme (meist im industriellen Umfeld) kann ein Bediener jedoch evtl. mehrere Anfragen gleichzeitig bearbeiten. Wenn ein Bediener stets mehrere Anfragen gleichzeitig bedienen soll, so kann dies durch einen Wert größer als 1 im Feld Kunden pro Bedienung festgelegt werden. Ist hier ein Wert größer als 1 hinterlegt und trifft ein einzelner Kunde in einem ansonsten leeren System ein, so muss dieser warten, bis genug weitere Kunden eingetroffen sind, um die geforderte Bedien-Batch-Größe zu erreichen.

Über den Verteilungseditor für die Bedienzeitenverteilung kann eingestellt werden, wie lange die einzelnen Bedienprozesse dauern sollen. In der Praxis sind hier häufig Log-Normalverteilungen oder Gamma-Verteilungen zur Modellierung von Gesprächszeiten sinnvoll.

Nach einem Gespräch kann es nötig sein, den Kunden zu einem weiteren Agenten zu leiten, um sein Anliegen abschließend zu behandeln. Welcher Anteil der bedienten Anrufer nach einem Gespräch weitergeleitet werden soll, kann über das Eingabefeld Weiterleitungswahrscheinlichkeit festgelegt werden.

Nachbearbeitungszeiten

Auf der Seite "Nachbearbeitungszeiten" kann eine Verteilung für die Zeitdauer, die ein Agent nach einem Gespräch mit einem Kunden benötigt, bevor er sich wieder als verfügbar meldet, eingestellt werden.

Der Kunde hat das System bereits verlassen oder wurde bereits weitergeleitet, wenn der Agent mit der Nachbearbeitung beginnt. Das bedeutet, dass die Nachbearbeitungszeit nicht zur Bedienzeit gezählt werden kann. Auf der anderen Seite ist der Agent jedoch auch während der Nachbearbeitungszeit noch durch die mit dem Kunden verbundenen Arbeiten belegt, so dass er in dieser Zeit noch nicht wieder für neue Anrufe zur Verfügung steht, d.h. die Nachbearbeitungszeit kann auch nicht als Bereitzeit des Agenten gezählt werden.

Wiederholungen

Ein Kunde, der entweder aufgrund der Tatsache, dass alle Warteplätze belegt waren, abgewiesen wurde oder aber aufgrund einer zu kurzen individuellen Wartezeittoleranz das Warten vorzeitig aufgegeben hat, kann später einen weiteren Anrufversuch starten. Über das Eingabefeld Wiederholwahrscheinlichkeit kann festgelegt werden, mit welcher Wahrscheinlichkeit dies auftritt.

Wenn ein Kunden sich entschließt, später einen weiteren Anrufversuch zu starten, so wartet er eine gewisse Zeit, bis er den weiteren Anrufversuch tätigt. Über den Verteilungseditor zur Festlegung der Wiederholabstände lässt sich einstellen, gemäß welcher Wahrscheinlichkeitsverteilung die individuellen Wiederholabstände festgelegt werden sollen.
